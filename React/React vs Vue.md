# React vs Vue

## Краткий ответ для собеседования

**React** — библиотека для UI с декларативным подходом, однонаправленным потоком данных и обновлением через Virtual DOM. **Vue** — фреймворк с реактивной моделью данных, шаблонами и «реактивностью из коробки». Отличия: React даёт больше свободы и экосистемы, Vue — быстрее старт и встроенные решения (роутинг, стейт). Под капотом: у React — реконсиляция и Fiber; у Vue — реактивный прокси и точечные обновления зависимостей.

---

## Основные отличия

### Философия и экосистема

**React** — библиотека только для UI. Роутинг, стейт-менеджмент, формы — из экосистемы (React Router, Redux, Zustand и т.д.). Гибко, но нужно выбирать и стыковать инструменты.

**Vue** — фреймворг «всё в одном»: официальный роутер, Pinia/Vuex, шаблоны, директивы. Меньше решений «как склеить», быстрее старт для типовых проектов.

### Модель данных и реактивность

**React** — явное обновление: меняешь состояние через setState/useState, компонент ререндерится. Реактивности «из коробки» нет: что обновить и когда — решает разработчик и реконсилятор.

**Vue** — реактивная модель: данные в data/composition API отслеживаются, при изменении обновляются только зависимые части. Меньше ручного контроля, но нужно понимать, как Vue отслеживает зависимости.

### Шаблоны и JSX

**React** — логика и разметка в JS (JSX). Всё на одном языке, условия и циклы — обычный JavaScript.

**Vue** — шаблоны в HTML-подобном синтаксисе с директивами (v-if, v-for, v-model). Отдельный слой шаблонов, но многим проще верстать и читать.

### Кривая обучения

**React** — меньше «магии», зато больше концепций: хуки, ререндеры, мемоизация, когда что ререндерить. Новичкам проще начать с малого, сложнее довести до уверенного уровня.

**Vue** — проще войти: шаблоны, реактивность и документация ориентированы на постепенное усложнение. Продвинутые темы (композиция, производительность) тоже есть.

---

## Плюсы и минусы React

**Плюсы:** огромная экосистема и сообщество, много вакансий, гибкость в выборе стека, сильная типизация с TypeScript, одна модель (компонент + состояние) для веба и мобильных (React Native).

**Минусы:** много решений на стороне разработчика (стейт, роутинг, формы), ререндеры и оптимизация требуют понимания (memo, useCallback, useMemo), для SEO и «быстрой первой отрисовки» часто нужен SSR/фреймворк (Next.js и т.д.).

---

## Плюсы и минусы Vue

**Плюсы:** быстрый старт, встроенные роутер и стейт, реактивность из коробки, шаблоны и директивы удобны для верстки, хорошая документация, меньше «подводных камней» с ререндерами.

**Минусы:** экосистема и рынок меньше, чем у React, продвинутая реактивность (ref, reactive, watchers) всё равно требует понимания, при сложной логике шаблоны могут разрастаться — тогда ближе к JSX/логике в коде.

---

## Под капотом (упрощённо)

### React

Дерево компонентов описывает, каким должен быть UI. При изменении состояния React строит новое дерево (Virtual DOM), сравнивает со старым (reconciliation) и вносит минимальные изменения в реальный DOM. С версии 16 задействован Fiber: работа разбита на фазы (render и commit), возможны приоритеты и прерывание рендера. Обновление всегда «сверху вниз» от изменившегося компонента — явного списка зависимостей от данных нет.

### Vue

Vue при инициализации оборачивает данные в реактивный слой (в Vue 3 — Proxy). При чтении полей во время рендера запоминаются зависимости «компонент — поле». При изменении поля обновляются только компоненты, которые от него зависят. Нет полного сравнения всего дерева: обновления точечные. Шаблон компилируется в render-функцию, оптимизации (статичные узлы, поднятие и т.д.) делаются на этапе компиляции.

---

## Когда что выбирать

**React** — большие команды, разнородный стек, нужна максимальная гибкость и экосистема, типизация и переиспользование логики между вебом и мобильными.

**Vue** — быстрый старт, много форм и классического CRUD, желание меньше думать о ререндерах и иметь «всё из коробки» в одном фреймворке.

## Вопросы на собеседовании

| Вопрос | Краткий ответ |
|--------|----------------|
| Чем React отличается от Vue? | React — библиотека UI, явное состояние, JSX; Vue — фреймворг, реактивность из коробки, шаблоны и директивы. |
| Почему Vue называют реактивным? | Данные отслеживаются, при изменении автоматически обновляются только зависимые части UI. |
| Почему React не реактивный? | Обновление инициируется явно (setState); реконсилятор сравнивает деревья, а не подписывается на поля данных. |
| Что быстрее — React или Vue? | Зависит от сценария; у Vue точечные обновления, у React — сравнение дерева и оптимизации (Fiber, мемо). |
| Где больше экосистема? | У React — больше библиотек, решений и вакансий; у Vue — компактный официальный стек. |

---

## Кратко для ответа

React — библиотека с явным состоянием и Virtual DOM; Vue — фреймворг с реактивной моделью и шаблонами. React гибче и с большей экосистемой; Vue проще войти и даёт роутинг/стейт из коробки. Под капотом у React — реконсиляция и Fiber; у Vue — реактивный прокси и точечное обновление зависимостей.
