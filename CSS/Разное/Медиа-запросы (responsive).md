# Медиа-запросы (responsive)

## Краткий ответ для собеседования

**Медиа-запросы** (@media) позволяют применять CSS **в зависимости от условий**: ширина экрана (viewport), тип устройства (screen, print), ориентация (orientation: portrait/landscape), плотность пикселей (resolution) и др. Типичное использование — **адаптивная вёрстка**: разные стили для мобильных, планшетов и десктопа по **min-width** / **max-width** (например, @media (max-width: 768px) { ... }). На собеседовании часто спрашивают: зачем медиа-запросы, что такое breakpoint, mobile-first vs desktop-first, какие условия бывают.

---

## Синтаксис и типичные условия

**@media** — правило с условием. Стили внутри применяются только если условие истинно.

Типичные условия:
- **min-width**, **max-width** — ширина viewport (чаще всего в px или em). Используют для breakpoint’ов: до 768px — мобильный, от 768px — планшет и т.д.
- **min-height**, **max-height** — высота viewport.
- **orientation: portrait** / **landscape** — ориентация устройства.
- **screen**, **print** — тип носителя (экран или печать).
- **prefers-reduced-motion** — предпочтение пользователя уменьшить анимацию (доступность).
- **resolution** — плотность пикселей (для ретины).

Пример: `@media (max-width: 768px) { .sidebar { display: none; } }` — на узком экране скрыть сайдбар.

---

## Mobile-first и desktop-first

- **Mobile-first**: базовые стили пишут для **малого** экрана; затем через **min-width** добавляют стили для больших (например, @media (min-width: 768px)). Проще наращивать, а не «убирать» лишнее на мобильных.
- **Desktop-first**: базовые стили для **большого** экрана; через **max-width** переопределяют для малых. Логика «упрощаем под мобильные».

На собеседовании можно сказать: «Mobile-first — сначала стили для мобильных, потом через min-width для больших экранов; часто удобнее и меньше переопределений.»

## Вопросы на собеседовании

| Вопрос | Краткий ответ |
|--------|----------------|
| Зачем медиа-запросы? | Применять разные стили в зависимости от размера экрана, типа устройства, ориентации — основа адаптивной вёрстки. |
| Что такое breakpoint? | Значение ширины (или другого параметра), при котором меняется раскладка/стили (например, 768px — переход от мобильного к планшету). |
| Что такое mobile-first? | Сначала стили для мобильных, затем через @media (min-width: ...) — для больших экранов; наращивание от малого к большому. |
| Какие условия в @media бывают? | min-width, max-width, orientation, screen/print, prefers-reduced-motion, resolution и др. |

---

## Кратко запомнить

**Медиа-запросы** (@media) — условное применение CSS по ширине/высоте viewport, типу носителя, ориентации. **Breakpoint** — порог (часто ширина в px), при котором меняется вёрстка. **Mobile-first** — базовые стили для малого экрана, затем min-width для больших. Основа responsive design.
