# Формы: \<form\>, \<input\>, \<label\>, \<button\>

## Краткий ответ для собеседования

**\<form\>** собирает данные и отправляет их на сервер (атрибуты **action**, **method**). **\<input\>** — поле ввода; тип задаётся **type** (text, email, password, checkbox, radio, submit и т.д.); **name** обязателен для отправки (пара имя=значение). **\<label\>** связывается с полем через **for** и **id** элемента — улучшает доступность и расширяет область клика. **\<button\>** — кнопка; **type**: submit (отправка формы), button (обычная кнопка), reset (сброс). На собеседовании могут спросить: зачем **name** у input, разницу между **id** и **name**, зачем **label** и как связать с полем, почему у кнопки внутри form важно указывать **type** (иначе в некоторых браузерах submit по умолчанию).

---

## \<form\>

- **action** — URL, на который отправляются данные (если пусто — текущая страница).
- **method** — GET (данные в URL) или POST (данные в теле запроса).
- **enctype** — способ кодирования (по умолчанию application/x-www-form-urlencoded; для файлов — multipart/form-data).

При отправке (submit) браузер собирает все поля с атрибутом **name** внутри формы и отправляет их как пары имя=значение.

---

## \<input\>

- **type** — text, email, password, number, checkbox, radio, file, submit, button, hidden и др.
- **name** — ключ при отправке формы; без name поле не попадёт в данные.
- **id** — уникальный идентификатор; используется для связи с **\<label for="...">** и в JS/CSS.
- **value** — значение (для кнопок — подпись; для полей — начальное значение).
- **required**, **disabled**, **readonly**, **placeholder**, **autocomplete**.

**Важно:** **id** — один на странице; **name** — может повторяться (например, у radio с одним именем — одна группа выбора). При отправке формы уходит **name**, не id.

---

## \<label\>

Связь с полем: явная (у **label** атрибут **for** равен **id** поля) или неявная (поле внутри \<label\>). Без label поля менее доступны и кликабельная область меньше.

---

## \<button\>

- **type="submit"** — отправляет форму (по умолчанию у button внутри form в части браузеров).
- **type="button"** — обычная кнопка, форму не отправляет.
- **type="reset"** — сбрасывает поля формы.

Всегда лучше задавать **type** явно.

---

## Вопросы на собеседовании

| Вопрос | Краткий ответ |
|--------|----------------|
| Зачем у input атрибут name? | Чтобы поле попало в данные формы при отправке; name — ключ в паре имя=значение. |
| В чём разница между id и name у input? | id — уникальный идентификатор для label и JS/CSS; name — ключ при отправке формы, у radio может совпадать у группы. |
| Зачем label и как связать с полем? | Доступность и расширение области клика; связь: label for="id" и у поля id="id", или поле внутри label. |
| Какие типы у button и зачем их указывать? | submit — отправить форму, button — без отправки, reset — сброс; указывать явно, чтобы не было неожиданной отправки. |
| Что такое enctype у form? | Способ кодирования данных; для загрузки файлов нужен multipart/form-data. |

---

## Кратко для ответа

**form** — action, method, enctype; отправляет поля с **name**. **input** — type, name (обязателен для отправки), id (для label и JS). **label** — for + id поля или обёртка вокруг поля. **button** — type submit/button/reset; лучше задавать явно. **id** уникален на странице; **name** — ключ в данных формы, у radio один name = одна группа.
