# Теги \<map\> и \<area\>

## Краткий ответ для собеседования

**\<map\>** задаёт **карту-изображение (image map)**: на одном изображении несколько кликабельных областей с разными ссылками или действиями. Связь с картинкой через атрибут **name** у \<map\> и **usemap="#..."** у \<img\>. Внутри \<map\> лежат элементы **\<area\>**: каждый задаёт форму области (**shape**: rect, circle, poly) и координаты (**coords**), а также **href** (куда ведёт клик), **alt** (обязателен для доступности). На собеседовании могут спросить: зачем нужны map/area, как связать map с img, какие бывают shape, зачем alt у area.

---

## Что такое карта-изображение

Одно изображение разбивается на **зоны**; у каждой зоны свой URL или поведение. Типичный пример: план помещения (клик по комнате → страница комнаты), географическая карта, диаграмма с переходами. Вместо нарезки на отдельные картинки и наложения ссылок через абсолютное позиционирование используется один тег \<img\> и описание областей в \<map\> и \<area\>.

---

## Связь map и img

- У **\<map\>** задаётся атрибут **name** (например, `name="floorplan"`).
- У **\<img\>** задаётся атрибут **usemap="#floorplan"** (значение = # + name карты).
- \<map\> может находиться в любом месте документа; связь идёт по имени.

Один \<map\> используется одним \<img\> (по usemap). Одно изображение может иметь только одну карту.

---

## Элемент \<area\>

Внутри \<map\> перечисляются элементы **\<area\>**. Каждый \<area\> описывает одну кликабельную область.

| Атрибут | Описание |
|--------|----------|
| **shape** | Форма: **rect** (прямоугольник), **circle** (круг), **poly** (многоугольник), **default** (вся оставшаяся область). |
| **coords** | Координаты в пикселях. rect: x1,y1,x2,y2 (левый верхний и правый нижний угол). circle: x,y,r (центр и радиус). poly: x1,y1,x2,y2,... (вершины по порядку). |
| **href** | URL при клике (если нет — область не ведёт по ссылке). |
| **alt** | Обязателен для доступности: описание области для скринридеров и при отсутствии href. |
| **target** | Как у ссылки (например, _blank). |

Координаты задаются относительно **изображения** (левый верхний угол — 0,0). При изменении размера картинки через CSS координаты не пересчитываются автоматически — возможны расхождения на ресайзе.

---

## Пример (прямоугольная область)

- Карта: \<map name="example"\>
- Область: \<area shape="rect" coords="0,0,100,50" href="/page1" alt="Перейти в раздел 1"\>
- Картинка: \<img src="image.png" alt="План" usemap="#example"\>

---

## Когда использовать

**Подходит для:** планов, карт, схем, диаграмм — одно изображение с несколькими логическими зонами и разными переходами. Когда областей мало и они простой формы (прямоугольники, круги), map/area достаточно.

**Не подходит для:** сложной адаптивной вёрстки (координаты фиксированы) и большого числа областей произвольной формы (удобнее накладывать невидимые ссылки или использовать SVG с \<a\> внутри).

---

## Доступность

У каждого \<area\> должен быть **alt** с описанием назначения области. Если область не ведёт по ссылке (href отсутствует), alt всё равно нужен. Так скринридеры и пользователи без мыши понимают, что за зона.

## Вопросы на собеседовании

| Вопрос | Краткий ответ |
|--------|----------------|
| Для чего нужны map и area? | Одно изображение с несколькими кликабельными областями; у каждой области свой href или действие. |
| Как связать map с img? | У map задать name, у img — usemap="#name". |
| Какие бывают shape у area? | rect (прямоугольник), circle (круг), poly (многоугольник), default (остальная область). |
| Что такое coords? | Координаты области в пикселях относительно изображения; формат зависит от shape. |
| Зачем alt у area? | Доступность: описание области для скринридеров; обязателен. |
| Минус map/area при адаптиве? | Координаты фиксированы, при изменении размера img зоны могут не совпадать; для сложной адаптивности часто используют SVG или другие приёмы. |

## Кратко для ответа

**\<map\>** и **\<area\>** — карта-изображение: несколько кликабельных зон на одной картинке. Связь: **name** у map, **usemap="#name"** у img. **\<area\>** задаёт shape (rect, circle, poly), coords и href; **alt** у area обязателен. Координаты в пикселях относительно изображения; при ресайзе картинки могут расходиться.
