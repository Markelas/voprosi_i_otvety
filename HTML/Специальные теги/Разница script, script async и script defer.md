Разница в том, **когда** браузер загружает и выполняет скрипт относительно парсинга HTML.

1. **\<script\>** (без атрибутов) — загрузка сразу при встрече в HTML. Выполнение сразу после загрузки, **останавливает парсинг HTML** до завершения выполнения скрипта. Подходит для критически важных inline-скриптов.

2. **\<script async\>** — загрузка **параллельно** с парсингом HTML. Выполнение **сразу после загрузки**, в момент выполнения парсинг приостанавливается. Порядок выполнения **не гарантирован** (если несколько async-скриптов). Используется для скриптов, которые не зависят от DOM и других скриптов (аналитика, реклама).

3. **\<script defer\>** — загрузка **параллельно** с парсингом HTML. Выполнение **после завершения парсинга HTML**, перед событием DOMContentLoaded. **Порядок выполнения сохраняется** (если несколько defer-скриптов). Подходит для скриптов, которым нужен полностью готовый DOM, но которые не должны блокировать загрузку страницы.
