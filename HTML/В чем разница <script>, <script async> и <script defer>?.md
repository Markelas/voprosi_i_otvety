
Разница в том, когда браузер загружает и выполняет скрипт относительно парсинга HTML.  
1. `<script>` (без атрибутов) - Загрузка: _сразу при встрече в_ _HTML_. Выполнение: сразу после загрузки, останавливает парсинг HTML до завершения выполнения скрипта_. Подходит для скриптов, которые должны выполняться до построения страницы (например, критически важные inline-скрипты).  
   
2. `<script async>` Загрузка: _параллельно с парсингом_ _HTML_. Выполнение: _сразу после загрузки, останавливает парсинг в момент выполнения_. Порядок выполнения не гарантирован (если несколько async-скриптов). _Используется для скриптов, которые не зависят от_ _DOM_ и других скриптов (например, аналитика, реклама).  
   
3. `<script defer>` Загрузка: _параллельно с парсингом_ _HTML_. Выполнение_: после завершения парсинга_ _HTML, перед событием DOMContentLoaded_. Порядок выполнения сохраняется (если несколько defer-скриптов). Подходит для скриптов, которым нужен полностью готовый DOM, но которые не должны блокировать загрузку страницы.
