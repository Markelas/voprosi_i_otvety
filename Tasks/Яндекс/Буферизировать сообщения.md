**Задание:**

Необходимо реализовать функцию `solution`, которая подключается к серверу и отображает сообщения в правильном порядке.

Условия:

1. Сообщения приходят от сервера через функцию `connect`, которая принимает колбэк `onMessage`. Этот колбэк вызывается каждый раз, когда приходит новое сообщение.
2. Формат сообщения:
```js
interface Message {
  id: number;    // id первого сообщения = 1, у каждого следующего на 1 больше
  text: string;
}
```

3. **Проблема:** сервер не гарантирует порядок сообщений. Они могут приходить в любом порядке (например: 7, 1, 2, 3, 6, 5, 4, 8).
4. **Задача:** отображать сообщения с помощью функции `render(message)` строго в правильном порядке (по возрастанию id), без пропусков, но при этом **максимально быстро**.

```js
function solution(connect, render) {
  // Ожидаемый следующий id (начинаем с 1)
  let nextExpectedId = 1;
  
  // Буфер для сообщений, которые пришли раньше времени
  const buffer = new Map();

  // Подключаемся к серверу
  connect(function onMessage(message) {
    // Сохраняем сообщение в буфер
    buffer.set(message.id, message);

    // Пока в буфере есть следующее ожидаемое сообщение — отрисовываем
    while (buffer.has(nextExpectedId)) {
      const msg = buffer.get(nextExpectedId);
      render(msg);
      buffer.delete(nextExpectedId);
      nextExpectedId++;
    }
  });
}
```

1. **`nextExpectedId`** — хранит id следующего сообщения, которое нужно отрисовать (начинаем с 1)
2. **`buffer`** — Map, куда складываем все приходящие сообщения
3. **При каждом новом сообщении**:
    - Кладём его в буфер
    - Проверяем, есть ли в буфере сообщение с `nextExpectedId`
    - Если есть — отрисовываем и увеличиваем счётчик
    - Повторяем, пока в буфере есть следующие по порядку

```text
Приходит: 7   1   2   3   6   5   4   8
Рисуем:   .   1   2   3   .   .   4,5,6,7   8
```

Точка означает, что ничего не рисуем, потому что ещё не пришло нужное сообщение
